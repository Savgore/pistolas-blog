Style Direction v1.1 — Hunter Palette — Mar 2026

pistolas.co.uk

The canonical design reference for the personal blog of Savva Pistolas. All colour values, typographic choices, spacing decisions, and component patterns are defined here. This document also serves as the seed CSS for the Eleventy build.

01

Design Principles

Content is the design

The typography, spacing, and colour exist entirely in service of readability. No decorative elements that don't carry information. Every choice should disappear into the reading experience.

Intentioned design, not minimalism

This is not a minimal site — it is a considered one. Decorative and visual elements are welcome, particularly on the homepage, provided they are elegant and earn their place. The question is never "is this minimal?" but "does this belong?" Stock filler and visual noise are refused; meaningful illustration, diagrams, and imagery are embraced.

Honest and personal

No marketing language. No "hero" photography. No fake CTAs. The writing is the product. The design signals that the author has taste and ideas, not that they're selling something.

Multi-stream without confusion

Security, systems, and other topics coexist. The stream tag is the only signal needed — no separate visual identities per topic. A reader following everything should feel coherence, not category chaos.

Performance is a value

No JavaScript frameworks. No tracking pixels. No cookie banners. Three Google Font families loaded once. Everything else is HTML and CSS. Fast everywhere, including on mobile data.

02

Colour Palette — Hunter

Named palette: Hunter
Background --color-bg · #F3F5F2 Cool near-white with a faint green breath. Primary page background. Crisper than parchment, never clinical.
Background Alt --color-bg-alt · #E6EAE5 Article sections, code blocks, tag chips. A shade deeper — sage-grey.
Rule --color-rule · #C2CAC2 All borders and dividing lines. Cool grey-green, consistent with the background's undertone.
Ink Muted --color-ink-muted · #4A564A Metadata, dates, captions, nav links, secondary prose. Forest grey.
Accent --color-accent · #2B5C38 Links, stream tags (active), post tags, blockquote rule, inline code. Deep bottle green.
Ink --color-ink · #161C15 Primary body text. Near-black with a trace of forest in it. Never pure #000.
Palette rationale

The Hunter palette sits in the deep forest end of the green spectrum — confident, grounded, and slightly botanical. The background has a faint green breath rather than a warm yellow cast, which reads as crisper and more precise than parchment. The accent is a true bottle green: saturated enough to carry real weight as a link colour and active tag, restrained enough not to shout. Chosen over olive (too antiquarian, too close to mustard) and orange/copper (too warm, too close to Anthropic's brand identity). The palette has associations with field guides, Ordnance Survey maps, and military technical documentation — all appropriate registers for someone writing about security, systems, and rigorous thinking.

Typographic rules
Display font usage

Playfair Display for all headings: hero h1, article titles, post list titles, section h2/h3 within articles. Never used for body copy or UI labels.

Mono font usage

Source Code Pro for all UI chrome: navigation links, stream tags, post dates, article metadata (date, read time), inline code, and code blocks. All nav/tag text is uppercase with wide tracking.

Italic as accent

Playfair's italic is beautiful — use it with intent. Hero h1 italicises one key word (often a verb or concept). Article titles may italic a phrase. Lora italic used for blockquotes and subtitles only.

Letter spacing rules

Display text: tight (–0.025em). Body: 0. Mono UI labels: wide (+0.10em). Mono tags/dates: very wide (+0.14em). Never letter-space body text or Lora prose.

04

Spacing & Layout

Key layout values
--max-content 680px Maximum width for article body prose. Enforces comfortable line length (~70–75 chars at 1.05rem).
--max-site 900px Maximum width for the overall site chrome — nav, hero, post list. Keeps the site feeling focused on any screen.
Spacing philosophy
Generous vertical rhythm

The site breathes. Post items get 26px top/bottom padding. Hero gets 64px top. Sections are separated by borders, not extra margin. Whitespace is structural, not decorative.

Consistent horizontal gutters

All content areas use 40px (--space-10) left/right padding on desktop, 20px on mobile. Nav, hero, stream bar, post list, and article all align to the same left edge.

Why Cyber Essentials Plus still gets the basics wrong

The framework is well-intentioned, but evidence requirements create perverse incentives for organisations already doing good security.

There's a version of Cyber Essentials that would make a real difference to small organisations' security posture. The one that exists is not quite that. After running a dozen CE+ assessments this year, a pattern becomes hard to ignore.

The problem isn't with the controls themselves — patch management, MFA, network boundaries. The problem is in what counts as evidence.

When you ask an assessor what adequate_firewall_config means in practice, you often get a different answer than the last assessor gave. This isn't incompetence — it's structural ambiguity baked into the scheme. The documentation gestures at intent without specifying method.

The evidence problem

Most CE+ assessors will accept a screenshot of a settings panel as evidence of a control being in place. Some will want a policy document. A few will want both. Almost none will ask whether the control is actually doing what it's supposed to do.

The stream field drives the filter bar. The description appears as the article subtitle and in OG meta tags. Always write a description — it's your SEO and social preview in one.

Writing conventions
Date format

Always "27 Feb 2026" — not ISO, not US format. Readable, unambiguous, and slightly European.

Post titles

Sentence case, not Title Case. Titles are thoughts, not headlines. Italicise one key concept where Playfair's italic can do work.

Stream names

Security · Systems · Now · Making. Capitalised, single word where possible. "Now" is for short notes, discoveries, and TILs. Streams are fixed — resist creating new ones.

Image conventions

Every post should include an og_image frontmatter field pointing to a social thumbnail (1200×630px). In-post diagrams live in /assets/posts/[slug]/. Images use descriptive alt text always. No captions unless the image genuinely requires one.

07

Anti-patterns — What to Avoid

Images must be intentional

Images are expected and welcome: every post should have an OG/social thumbnail, diagrams and illustrations are encouraged where they add to the argument, and the homepage may carry decorative visual content where it is elegant and meaningful. No stock photography, no decorative filler. If it doesn't earn its space, it doesn't ship.

No JavaScript

The site ships zero JS to the browser (search can be the exception, implemented as a static JSON index). No analytics, no comment widgets, no share buttons.

No font mixing

Three font families, fixed roles, no exceptions. Do not use Playfair for body text at any size. Do not use Lora for navigation. Do not introduce a fourth family.

No colour additions

The 6 palette colours plus pure white (for the label bars) are the only colours used. No additional accents, no per-stream colour coding, no gradient experiments.

No rounded corners

Everything is square-edged. Borders are straight lines. This is a document aesthetic, not an app aesthetic. Border-radius: 0 everywhere.

No hover animations

Colour transitions on links and tags only (0.15s ease). No transforms, no scale effects, no box-shadow reveals. Motion should be invisible, not decorative.

08

Visual Artefacts & Image Design

Philosophy

All visual artefacts — OG thumbnails, in-post diagrams, homepage illustrations — must feel like they belong to the same object as the site. An image that looks like it was generated by a different tool, designed for a different brand, or sourced from a stock library will rupture the reading experience. The palette, type, and spatial logic of the site extend into every image.

The single rule

If an image were placed next to the site's nav bar, would it look like it came from the same hand? If not, it doesn't ship. Palette consistency is non-negotiable. Typographic consistency is strongly preferred.

Palette application in images

Images draw from the same six-token palette as the site. Photography is the exception (it carries its own colours) but should be processed to feel tonally warm rather than cool or oversaturated. Diagrams, illustrations, and thumbnails are created entirely within the palette.

Generating with AI tools

OG thumbnails are typography-first and can be generated programmatically. A Satori (Vercel) or sharp-based Node script at build time can produce every thumbnail automatically from frontmatter — no manual Figma work needed. The 11ty build pipeline can call this as a transform.

Generating manually (Figma / Sketch)

Use the exact hex values from this document. Import Playfair Display and Source Code Pro from Google Fonts. Start from a template frame — do not design each one from scratch. The template has the top rule, corner decoration, and type hierarchy locked; only the title text and stream tag change per post.

Generating with image AI (Midjourney / DALL-E)

Not recommended for thumbnails — AI image tools cannot reliably reproduce the palette or typeset Playfair Display. Use them only for illustrative in-post images where photographic or illustrative content is genuinely needed, then place those images within a consistent framing (border rule, caption) defined in the site CSS.

In-post image guidelines
Diagrams & technical figures

Always SVG where possible. Draw using only palette colours. Strokes: 1px --color-rule for structure, 1–2px --color-accent for emphasis. Fill: --color-bg or --color-bg-alt. Labels: Source Code Pro, uppercase. Figure caption: Lora italic below in --color-ink-muted. Export at 2× for retina.

Photographs & screenshots

Screenshots should be taken on a light OS theme with window chrome removed where possible. Crop tightly. Apply a 1px --color-rule border in CSS (not baked into the image). Photographs should be cool-to-neutral toned; heavily warm or saturated photos can clash with the sage-grey background. No drop shadows.

Homepage decorative images

The homepage may use a single decorative visual element: a geometric illustration, an abstract mark, or a considered typographic composition. It must use only palette colours. It should feel like it was drawn specifically for this site. Size: contained within the hero area, never full-bleed.

Quick reference — palette hex values for image creation

Copy these directly into Figma, Illustrator, or any image generation prompt. When prompting AI image tools, describe the palette as: "cool near-white background (#F3F5F2), deep bottle green accents (#2B5C38), near-black ink (#161C15), crisp sage-grey tones — no warm yellows, no saturated colours, no pure white backgrounds."
